#### Question

A supported source database has change data capture (CDC) enabled. A Fabric load must incrementally synchronize inserts, updates, and deletes to its destination and resume from the last successful state after a failure. Which configuration should you choose?

#### Choices

- A. An incremental Copy job that uses the detected CDC feed
- B. A watermark-based Copy job on an increasing integer column
- C. A full-copy Copy job that appends every source row on each run
- D. A OneLake shortcut to the relational table

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. An incremental Copy job that uses the detected CDC feed**

#### Explanation

For a supported CDC-enabled source, Copy job detects the change feed and transfers inserts, updates, and deletes since the last successful run. Copy job manages that successful-run state, so a failed run resumes without advancing the checkpoint.

#### Why the other choices are incorrect

- **A:** CDC satisfies the delete-capture requirement and provides managed incremental state.
- **B:** A watermark can identify inserts and updates when its value changes, but it cannot detect source deletions.
- **C:** Repeated full appends duplicate existing data and do not model source deletes.
- **D:** A shortcut is a reference to supported file or OneLake locations, not a CDC synchronization process for this database source.

</details>
