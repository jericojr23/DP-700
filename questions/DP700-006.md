#### Question

Several terabytes of Parquet files are already stored in an Azure Data Lake Storage Gen2 account with hierarchical namespace enabled. A Fabric lakehouse must expose the current files quickly while avoiding another physical copy. What should you create?

#### Choices

- A. An ADLS Gen2 shortcut in the lakehouse
- B. A full-copy Copy job
- C. A Dataflow Gen2 destination that replaces the files
- D. A deployment pipeline stage

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. An ADLS Gen2 shortcut in the lakehouse**

#### Explanation

A OneLake shortcut can reference a folder in an ADLS Gen2 account from a lakehouse. It makes the external data accessible through Fabric without moving or duplicating the original files.

#### Why the other choices are incorrect

- **A:** The shortcut meets both the rapid-access and zero-copy requirements.
- **B:** A full-copy job would create the additional physical copy the scenario prohibits.
- **C:** Loading to a Dataflow Gen2 destination moves transformed data rather than exposing the existing files in place.
- **D:** A deployment pipeline manages item lifecycle stages and does not provide access to external data.

</details>
