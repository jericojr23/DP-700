#### Question

A platform team must store and analyze billions of application log events, JSON telemetry records, and time-series metrics. Users need fast KQL exploration over structured, semistructured, and free-text data with near-real-time ingestion. Which Fabric item should you use?

#### Choices

- A. Eventhouse with a KQL database
- B. Warehouse
- C. Dataflow Gen2
- D. Shortcut

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Eventhouse with a KQL database**

#### Explanation

Eventhouses are designed for storing and analyzing streaming event data and can query billions of events quickly. They are tailored to time-based events, telemetry, logs, structured records, semistructured data such as JSON, and unstructured free-text analysis through KQL databases.

#### Why the other choices are incorrect

- **A:** Eventhouse with a KQL database directly matches high-volume event, telemetry, log, and KQL analytics requirements.
- **B:** Warehouse is SQL-first and suited to relational analytics models, not KQL-first streaming telemetry exploration.
- **C:** Dataflow Gen2 is for visual data preparation, not high-volume event storage and KQL querying.
- **D:** A shortcut references data in place but does not provide the event-optimized query and storage engine.

</details>
