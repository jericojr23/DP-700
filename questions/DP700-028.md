#### Question

A manufacturer wants to ingest live IoT Hub messages into Fabric, apply lightweight transformations, and route the stream to multiple destinations for storage and analytics. The team wants a no-code streaming ingestion experience with built-in source connectors. Which Fabric item should it create?

#### Choices

- A. Eventstream
- B. Warehouse
- C. Lakehouse shortcut
- D. Dataflow Gen2

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Eventstream**

#### Explanation

Fabric eventstreams bring real-time events into Fabric, can transform event data, and route the stream to supported destinations without writing code. Eventstreams include source connectors such as Azure Event Hubs and Azure IoT Hub for streaming ingestion.

#### Why the other choices are incorrect

- **A:** Eventstream provides the required no-code streaming ingestion, transformation, source connectors, and routing.
- **B:** Warehouse is for SQL analytics over structured data and is not the streaming ingestion component.
- **C:** A lakehouse shortcut references existing data without copying it; it does not ingest live events from IoT Hub.
- **D:** Dataflow Gen2 is a low-code batch preparation tool, not the event ingestion and routing service for live streams.

</details>
