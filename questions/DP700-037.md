#### Question

An eventstream has a new `GroupByStreet` transformation connected to the Bicycles sample stream. After the transformation is saved, the canvas shows an error because the transformed output has nowhere to go. The team needs to persist the grouped results for KQL analysis. What should they do?

#### Choices

- A. Add an Eventhouse destination from the `GroupByStreet` node, create the `bikes-by-street` destination table, save, and publish the eventstream
- B. Delete the `GroupByStreet` node and query the raw `bikes` table with `take 100`
- C. Add a Dataflow Gen2 item to refresh the transformed stream on a schedule
- D. Open the workspace settings page and change the workspace license mode before saving the transformation

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Add an Eventhouse destination from the `GroupByStreet` node, create the `bikes-by-street` destination table, save, and publish the eventstream**

#### Explanation

The lab notes that saving the Group by configuration leaves the eventstream in an error state because the transformation output must be stored somewhere. The documented resolution is to add a new Eventhouse node from the `GroupByStreet` node, create a destination table named `bikes-by-street`, save the destination, publish the eventstream, and wait for the changes to become active.

#### Why the other choices are incorrect

- **A:** This gives the transformed stream a supported Eventhouse destination and makes the grouped output available for KQL analysis.
- **B:** Removing the transformation avoids the error by abandoning the grouped output requirement.
- **C:** Dataflow Gen2 is not the lab's destination for a live Eventstream transformation and does not persist the eventstream node output for KQL analysis.
- **D:** The issue is the missing destination after the transformation, not the workspace license mode.
