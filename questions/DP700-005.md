#### Question

OneLake security is enabled on a lakehouse. A Microsoft Entra group must read only the `Files/curated/region-a` folder across Fabric experiences. Its users currently inherit broad lakehouse read access through the automatically created `DefaultReader` role. Which action best enforces the requirement?

#### Choices

- A. Add the group to a new OneLake security role scoped to the folder and remove its broad access through `DefaultReader` or `ReadAll`
- B. Assign the group the workspace Contributor role and share the folder with Read permission
- C. Apply row-level security to every table and leave `DefaultReader` unchanged
- D. Create a shortcut to the folder and assign the group the workspace Viewer role

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Add the group to a new OneLake security role scoped to the folder and remove its broad access through `DefaultReader` or `ReadAll`**

#### Explanation

OneLake security roles can grant access to specific folders. When the preview is enabled, existing readers can remain in `DefaultReader` through virtual membership based on `ReadAll`; that broader access must be removed or the folder-specific restriction is ineffective.

#### Why the other choices are incorrect

- **A:** This grants the required folder access and removes the existing path that exposes all lakehouse data.
- **B:** Contributor grants broad workspace authoring and OneLake access, violating least privilege.
- **C:** Row-level security applies to table rows, not an unstructured folder, and leaving `DefaultReader` preserves broad access.
- **D:** A shortcut references data but does not override the source security model; Viewer is not a substitute for a folder-scoped OneLake role.
