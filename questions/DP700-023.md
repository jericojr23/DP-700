#### Question

A data engineering team must parse nested JSON files, call a Python package for address standardization, join the results to reference Delta tables, and write curated Delta tables back to OneLake. The transformation must be maintained as code. Which Fabric item is the best fit?

#### Choices

- A. Fabric notebook
- B. Dataflow Gen2
- C. Warehouse query editor
- D. Eventstream

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Fabric notebook**

#### Explanation

Fabric notebooks provide a code-first Spark environment for Python, Scala, SQL, and R. They are suited for complex transformations, custom algorithms, external libraries, joins, windowed aggregations, and writing results back as Delta tables in OneLake.

#### Why the other choices are incorrect

- **A:** A notebook supports the required PySpark code, Python package usage, joins, and Delta output.
- **B:** Dataflow Gen2 is optimized for visual low-code transformation and is not the best match for custom Python package logic.
- **C:** The Warehouse query editor is SQL-focused and does not run the requested PySpark and Python-library transformation.
- **D:** Eventstream processes events in motion and is not intended for this code-first batch enrichment.
