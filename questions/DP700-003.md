#### Question

A notebook is promoted from development to test and then production by using a Fabric deployment pipeline. The same notebook code must run in every stage, but its default lakehouse must point to the lakehouse in the target stage after each deployment. What should you configure?

#### Choices

- A. A data source rule on the notebook
- B. A parameter rule on the notebook
- C. A default lakehouse deployment rule in each target stage
- D. A workspace Viewer role for the notebook owner

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**C. A default lakehouse deployment rule in each target stage**

#### Explanation

Fabric deployment pipelines support a default lakehouse rule specifically for notebooks. The rule is defined in the target stage and assigns that stage's lakehouse when content is deployed into it.

#### Why the other choices are incorrect

- **A:** Deployment-pipeline data source rules are not supported for notebooks; notebooks use default lakehouse rules.
- **B:** Parameter rules are not the supported rule type for assigning a notebook's default lakehouse.
- **C:** This rule preserves the notebook while binding it to the correct target-stage lakehouse.
- **D:** A workspace role controls a user's permissions and cannot change an item's stage-specific lakehouse binding.
