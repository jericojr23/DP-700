#### Question

A data engineering team must land raw Parquet and CSV files, keep unstructured files available, transform data with Spark, and publish curated Delta tables for SQL analytics without moving data between separate systems. Which Fabric item should be the primary storage and engineering workspace?

#### Choices

- A. Lakehouse
- B. Warehouse
- C. Eventhouse
- D. Data pipeline

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Lakehouse**

#### Explanation

A Fabric lakehouse combines data lake storage with warehouse-style querying. It stores structured and unstructured data in OneLake, manages tables with Delta Lake, and supports both Apache Spark and SQL access without moving data between systems.

#### Why the other choices are incorrect

- **A:** Lakehouse supports files, Delta tables, Spark transformations, and SQL analytics over one copy of data.
- **B:** Warehouse is better for SQL-first structured dimensional models, not mixed file storage and Spark-first engineering.
- **C:** Eventhouse is built for event-based telemetry, logs, and KQL analytics rather than general batch file and Delta table engineering.
- **D:** A data pipeline orchestrates work but is not the storage item for files and Delta tables.
