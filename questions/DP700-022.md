#### Question

A daily process must copy files from storage, run a Dataflow Gen2 transformation, execute a notebook only after the dataflow succeeds, and send a notification when the workflow completes. The team wants low-code control flow, scheduling, and monitoring for the full sequence. Which Fabric item should coordinate the process?

#### Choices

- A. Data pipeline
- B. Lakehouse
- C. Warehouse
- D. KQL database

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Data pipeline**

#### Explanation

Data Factory pipelines orchestrate multiple data movement, transformation, and control-flow activities in one workflow. They can include activities such as Copy, Dataflow Gen2, notebooks, conditions, loops, schedules, and event-based triggers, which matches the required dependency chain.

#### Why the other choices are incorrect

- **A:** A data pipeline is designed to orchestrate activities, dependencies, schedules, and workflow monitoring.
- **B:** A lakehouse stores and processes data, but it does not provide the requested activity sequencing and notification workflow by itself.
- **C:** A warehouse supports SQL analytics and dimensional modeling, not general low-code orchestration across Fabric activities.
- **D:** A KQL database stores and queries real-time analytics data; it is not the orchestration controller for batch workflows.

</details>
