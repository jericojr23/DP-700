#### Question

A Fabric eventstream groups bicycle observations by `Street` using a 5-second tumbling window and writes the transformed output to an Eventhouse table named `bikes-by-street`. The output includes `Street`, `SUM_No_Bikes`, and `Window_End_Time`. You need a KQL result that shows total bikes per street for each completed window, with the newest windows first. Which query should you use?

#### Choices

- A. `['bikes-by-street'] | summarize TotalBikes = sum(tolong(SUM_No_Bikes)) by Window_End_Time, Street | sort by Window_End_Time desc, Street asc`
- B. `bikes | summarize TotalBikes = count() by Street | sort by Street asc`
- C. `['bikes-by-street'] | where Window_End_Time == now() | project Street, No_Bikes`
- D. `bikes | make-series TotalBikes = sum(No_Bikes) on ingestion_time() from now(-5s) to now() step 1s`

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. `['bikes-by-street'] | summarize TotalBikes = sum(tolong(SUM_No_Bikes)) by Window_End_Time, Street | sort by Window_End_Time desc, Street asc`**

#### Explanation

The lab's Group by transformation calculates `SUM_No_Bikes` by `Street` in a 5-second tumbling window and writes the transformed records to `bikes-by-street`. The lab then queries that table by summing `SUM_No_Bikes` as a numeric value, grouping by `Window_End_Time` and `Street`, and sorting newest window end times first.

#### Why the other choices are incorrect

- **A:** This uses the transformed table and the window-ending timestamp produced by the Eventstream aggregation, then groups by the required window and street dimensions.
- **B:** This queries the raw `bikes` table and counts records, so it does not use the transformed per-window sum of available bikes.
- **C:** An exact equality to `now()` is not a reliable way to match completed tumbling windows, and the transformed output uses `SUM_No_Bikes` rather than raw `No_Bikes`.
- **D:** This builds a new KQL time series from the raw table instead of querying the Eventstream-generated grouped table that already contains the windowed aggregation output.
