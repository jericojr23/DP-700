#### Question

A lakehouse already contains governed Delta tables used by another team. Your team must expose a subset of those tables in its own lakehouse for analysis, but storage policy prohibits making another physical copy. Which Fabric feature should you create?

#### Choices

- A. OneLake shortcut
- B. Full-copy data pipeline
- C. Warehouse table created with CTAS
- D. Dataflow Gen2 destination table

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. OneLake shortcut**

#### Explanation

OneLake shortcuts let a lakehouse or KQL database reference existing OneLake data from another location. The shortcut appears in the target item while pointing to the source data, which avoids duplicating the underlying files or tables.

#### Why the other choices are incorrect

- **A:** A shortcut exposes existing data in place and satisfies the no-copy requirement.
- **B:** A full-copy pipeline creates the extra physical copy that the scenario prohibits.
- **C:** Creating a warehouse table with CTAS materializes another table rather than referencing the source data in place.
- **D:** A Dataflow Gen2 destination writes prepared output and would create another stored copy.
