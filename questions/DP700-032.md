#### Question

A data engineer has created an eventhouse and a KQL database for a bike-share monitoring solution. The team needs to use the built-in Bicycles sample stream, create a new KQL table named `bikes`, and preview the ingested events after the stream is active. Which Eventstream configuration should the engineer use?

#### Choices

- A. Add the Bicycles sample source, add an Eventhouse destination with event processing before ingestion, create the `bikes` destination table, save, and publish
- B. Create a warehouse table named `bikes`, then use a Dataflow Gen2 refresh to poll the Bicycles sample source
- C. Add a OneLake shortcut to the eventhouse, then type `bikes` directly into the destination table box without creating the table
- D. Create an Activator rule first, then use the rule history as the source table for KQL queries

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Add the Bicycles sample source, add an Eventhouse destination with event processing before ingestion, create the `bikes` destination table, save, and publish**

#### Explanation

The lab configures an eventstream with the Bicycles sample data source, then adds an Eventhouse destination using event processing before ingestion. The destination is mapped to the workspace eventhouse and KQL database, creates a new table named `bikes`, uses JSON input, and must be saved and published before the destination becomes active and data preview is available.

#### Why the other choices are incorrect

- **A:** This follows the Eventstream-to-Eventhouse ingestion path and creates the KQL destination table that receives the live sample events.
- **B:** Dataflow Gen2 is a batch-oriented transformation item and is not the lab path for streaming the Bicycles sample data into an Eventhouse table.
- **C:** A OneLake shortcut does not attach a live Eventstream source, and the lab warns to create the new KQL destination table through the create-new action rather than typing directly into the table box.
- **D:** Activator monitors events and triggers actions, but it is not used as the ingestion source table for KQL analysis in this Eventhouse loading pattern.
