#### Question

IoT telemetry arrives continuously from Event Hubs. Before storage, the solution must filter invalid events, remove duplicates, calculate five-minute windowed aggregates, and route different device categories to separate destinations. The team wants a no-code implementation. Which Fabric component should perform this processing?

#### Choices

- A. Eventstream
- B. Dataflow Gen2
- C. Deployment pipeline
- D. Warehouse SQL query editor

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Eventstream**

#### Explanation

Eventstreams collect, transform, and route real-time data without code. Their event-processing capabilities include filtering, cleansing, windowed aggregation, duplicate detection, and content-based routing to destinations.

#### Why the other choices are incorrect

- **A:** Eventstream natively combines the required no-code, streaming transformations and routing.
- **B:** Dataflow Gen2 is a Power Query data-preparation tool and is not the component designed for continuous event-in-motion processing and routing.
- **C:** A deployment pipeline manages application lifecycle promotion, not streaming records.
- **D:** The Warehouse SQL query editor queries stored data; it does not provide this no-code event-routing topology before storage.

</details>
