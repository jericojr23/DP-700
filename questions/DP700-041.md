#### Question

A security team is publishing a Real-Time Dashboard over a KQL database in an eventhouse. Each viewer must see query results only when their own permissions allow access to the underlying data. The dashboard must not let viewers query data by inheriting the dashboard creator's permissions. Which data source authorization option should the team choose?

#### Choices

- A. Pass-through identity
- B. Dashboard editor's identity
- C. Workspace Contributor role for every viewer
- D. A text tile that lists the dashboard owner

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Pass-through identity**

#### Explanation

Pass-through identity makes each dashboard viewer access the data source by using their own permissions. That matches the requirement to preserve per-user authorization and avoid letting viewers use the dashboard creator's data access.

#### Why the other choices are incorrect

- **A:** Pass-through identity evaluates data access with each viewer's own permissions, so it satisfies the security requirement.
- **B:** Dashboard editor's identity makes all viewers access the data source with the dashboard creator's permissions, which is exactly what the scenario prohibits.
- **C:** Granting every viewer Contributor would broaden workspace authoring permissions and still does not specifically configure the dashboard data source authorization mode.
- **D:** A text tile can add explanatory content to a dashboard, but it cannot enforce data source permissions.

</details>
