#### Question

A Fabric notebook loads product sales into a PySpark dataframe named `sales_df` with columns `Category` and `Revenue`. You need a new dataframe that contains one row per category and the average revenue for that category. Which expression should you use?

#### Choices

- A. `sales_df.groupBy("Category").agg({"Revenue": "avg"})`
- B. `sales_df.select("Category", "Revenue").where(sales_df["Revenue"] > 0)`
- C. `sales_df.agg({"Revenue": "avg"}).groupBy("Category")`
- D. `sales_df.write.partitionBy("Category").parquet("Files/sales_by_category")`

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. `sales_df.groupBy("Category").agg({"Revenue": "avg"})`**

#### Explanation

PySpark DataFrame transformations use `groupBy` to group rows by a column and aggregation functions to calculate values for each group. Grouping by `Category` before applying the average aggregation returns a dataframe with one aggregated result per category.

#### Why the other choices are incorrect

- **A:** This groups rows by category and calculates the average revenue for each group.
- **B:** This filters rows and keeps detail-level records; it does not aggregate revenue by category.
- **C:** This attempts to aggregate the entire dataframe before grouping, so it does not produce category-level averages.
- **D:** Partitioning output files by category changes storage layout but does not calculate an average value.

</details>
