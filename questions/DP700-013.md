#### Question

A published eventstream carries machine telemetry. Operations staff need a Microsoft Teams notification whenever an individual event reports `temperature > 90`, and they want to manage activation history and test the notification rule. What should you configure?

#### Choices

- A. Use **Set alert** on the live eventstream and create an Activator rule with an **On each event when** condition
- B. Create a scheduled Dataflow Gen2 refresh that filters the temperature column
- C. Add a deployment rule that replaces the temperature parameter
- D. Run `OPTIMIZE` on the destination Delta table after each event

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Use Set alert on the live eventstream and create an Activator rule with an On each event when condition**

#### Explanation

Setting an alert on a published, live eventstream automatically adds an Activator destination. The `On each event when` condition evaluates a field against a value, and Activator supports testing notifications and reviewing activation history.

#### Why the other choices are incorrect

- **A:** This directly evaluates each live event and provides the requested notification and rule-management features.
- **B:** A scheduled batch refresh does not evaluate each event as it arrives and is not the documented eventstream alert path.
- **C:** Deployment rules change stage-specific configuration during promotion; they do not monitor telemetry.
- **D:** Delta optimization changes file layout and cannot send threshold-based notifications.

</details>
