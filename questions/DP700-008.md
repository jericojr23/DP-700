#### Question

A data engineering team must transform large semi-structured datasets in OneLake. The solution requires reusable Python functions, a third-party library, and custom PySpark logic before writing Delta tables. Which Fabric item is the best fit for the transformation?

#### Choices

- A. Dataflow Gen2
- B. Fabric notebook
- C. Deployment pipeline
- D. Activator

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**B. Fabric notebook**

#### Explanation

Fabric notebooks provide a code-first Spark environment for advanced transformations and support Python and PySpark. That makes a notebook the appropriate choice for reusable code, libraries, and custom processing of OneLake data.

#### Why the other choices are incorrect

- **A:** Dataflow Gen2 is optimized for visual Power Query transformations and does not best meet the explicit custom PySpark and Python-library requirements.
- **B:** A notebook provides the requested code-first language, library, and distributed Spark capabilities.
- **C:** A deployment pipeline promotes items between environments; it does not execute the transformation logic described.
- **D:** Activator detects conditions and triggers actions; it is not a batch data-transformation engine.

</details>
