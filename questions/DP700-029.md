#### Question

A data engineering team already has a streaming source available in Real-Time Hub. The team creates a Fabric eventstream and needs to add that source to the eventstream without defining a brand-new source connection. Which action should the team take?

#### Choices

- A. Use the eventstream canvas to connect to the existing source from Real-Time Hub
- B. Create a OneLake shortcut to the Real-Time Hub source
- C. Create a Lakehouse destination before adding the source
- D. Use Fabric Activator to create the eventstream source

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Use the eventstream canvas to connect to the existing source from Real-Time Hub**

#### Explanation

After an eventstream is created, sources are added through the eventstream canvas. The unit states that you can either create a new source or connect to an existing source from Real-Time Hub, which directly matches the scenario.

#### Why the other choices are incorrect

- **A:** This uses the supported source configuration path and reuses the already available Real-Time Hub source.
- **B:** A OneLake shortcut references existing data locations; it is not the mechanism for attaching a streaming source to an eventstream.
- **C:** A destination receives processed event data after a source, and optional transformations, have been configured. It does not attach the existing source.
- **D:** Fabric Activator is a destination used to detect conditions and trigger actions; it does not create eventstream source connections.

</details>
