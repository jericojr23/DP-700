#### Question

A Real-Time Dashboard tile queries a `bikes` table that receives frequent bike-station observations. The tile must show one current row per `Neighbourhood`, using only records ingested in the last 30 minutes, and it must keep the values from the most recently ingested row in each neighborhood. Which KQL query best meets the requirement?

#### Choices

- A. `bikes | where ingestion_time() between (ago(30min) .. now()) | summarize latest_observation = arg_max(ingestion_time(), *) by Neighbourhood | project Neighbourhood, latest_observation, No_Bikes, No_Empty_Docks`
- B. `bikes | summarize count() by Neighbourhood | where ingestion_time() between (ago(30min) .. now())`
- C. `bikes | where ingestion_time() > now() | project Neighbourhood, No_Bikes, No_Empty_Docks`
- D. `bikes | summarize latest_observation = max(ingestion_time()) | project Neighbourhood, No_Bikes, No_Empty_Docks`

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. `bikes | where ingestion_time() between (ago(30min) .. now()) | summarize latest_observation = arg_max(ingestion_time(), *) by Neighbourhood | project Neighbourhood, latest_observation, No_Bikes, No_Empty_Docks`**

#### Explanation

The dashboard unit uses this pattern to filter the table to the last 30 minutes, group by `Neighbourhood`, and use `arg_max(ingestion_time(), *)` to keep the latest ingested row for each group. The projection then exposes the neighborhood and bike availability fields needed by the tile.

#### Why the other choices are incorrect

- **A:** This applies the ingestion-time window before summarizing and returns the latest full observation per neighborhood.
- **B:** This counts records instead of returning the latest observation values, and it places an ingestion-time filter after a summary that no longer has per-row ingestion timestamps.
- **C:** `ingestion_time() > now()` looks for records ingested in the future, so it does not return the last 30 minutes of data.
- **D:** This finds one maximum ingestion time across the full input and does not group by `Neighbourhood` or preserve the row values associated with each neighborhood's latest observation.
