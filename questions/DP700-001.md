#### Question

A data engineer must create and modify lakehouses, warehouses, notebooks, and pipelines in a Fabric workspace. The engineer must not add users, change workspace access, or update the workspace itself. Which workspace role should you assign?

#### Choices

- A. Admin
- B. Member
- C. Contributor
- D. Viewer

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**C. Contributor**

#### Explanation

The Contributor role can create and modify Fabric items and execute pipelines, but it cannot add or remove people or update the workspace. It therefore provides the required authoring capabilities without the access-management permissions excluded by the scenario.

#### Why the other choices are incorrect

- **A:** Admin can update or delete the workspace and manage all workspace access, which exceeds the required permissions.
- **B:** Member can add members and users with lower permissions, so it does not satisfy the access-management restriction.
- **C:** Contributor supports item creation and modification without granting workspace or membership administration.
- **D:** Viewer can read workspace content but cannot create or modify the required Fabric items.

</details>
