#### Question

A data engineering team runs several Fabric notebooks and Spark job definitions. The team must use the same Spark runtime, libraries, and compute defaults for every new session, and workspace admins want those settings managed as one reusable workspace item. What should you configure?

#### Choices

- A. Create and publish a Fabric environment, then attach the notebooks and Spark job definitions to it
- B. Add every engineer to the workspace Admin role
- C. Store the library list in a lakehouse Files folder and import it from each notebook
- D. Create a Dataflow Gen2 item and schedule it before each notebook run

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Create and publish a Fabric environment, then attach the notebooks and Spark job definitions to it**

#### Explanation

A Fabric environment is a workspace item for Spark session configuration. It can define the Spark runtime, compute settings, libraries, and small resources for notebooks and Spark job definitions. Publishing applies Spark compute and library changes, making the environment the reusable governed configuration the scenario requires.

#### Why the other choices are incorrect

- **A:** An environment directly centralizes the runtime, library, and Spark compute configuration for the affected Spark sessions.
- **B:** Workspace Admin grants broad administration permissions but does not create reusable Spark runtime or library defaults for jobs.
- **C:** A Files folder can store data or resources, but it does not govern Spark runtime selection, compute settings, or library publishing.
- **D:** Dataflow Gen2 is a transformation item and cannot define Spark session settings for notebooks or Spark job definitions.
