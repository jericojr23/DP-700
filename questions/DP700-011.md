#### Question

An operator needs one centralized Fabric page to search, sort, and filter pipeline runs across Data Factory items by status, start time, run kind, and location. Where should the operator go?

#### Choices

- A. Monitoring hub
- B. OneLake catalog
- C. Deployment pipelines
- D. Manage connections and gateways

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Monitoring hub**

#### Explanation

The Monitoring hub is the centralized portal for browsing pipeline runs across Fabric items. It supports searching and filtering runs and sorting columns such as status, start time, item type, location, and run kind.

#### Why the other choices are incorrect

- **A:** Monitoring hub supplies the requested cross-item operational view and run filters.
- **B:** OneLake catalog is used to discover data and Fabric items, not to inspect pipeline execution history.
- **C:** Deployment pipelines move content through lifecycle stages rather than monitor Data Factory runs.
- **D:** Connection and gateway settings manage connectivity but do not aggregate pipeline-run status.
