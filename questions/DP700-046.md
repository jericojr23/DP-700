#### Question

An Eventhouse table named `Telemetry` has columns `Timestamp`, `DeviceId`, and `Severity`. An operations dashboard needs a KQL query that returns the number of error events for each device in explicit five-minute time buckets, with the newest buckets first. Which query should you use?

#### Choices

- A. `Telemetry | where Severity == "Error" | summarize ErrorCount = count() by DeviceId, WindowStart = bin(Timestamp, 5m) | sort by WindowStart desc`
- B. `Telemetry | summarize ErrorCount = count() by DeviceId | where Severity == "Error" | sort by Timestamp desc`
- C. `Telemetry | where Severity == "Error" | project DeviceId, WindowStart = bin(Timestamp, 5m) | sort by WindowStart desc`
- D. `Telemetry | where Severity == "Error" | summarize ErrorCount = count() by Timestamp, DeviceId | sort by DeviceId asc`

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. `Telemetry | where Severity == "Error" | summarize ErrorCount = count() by DeviceId, WindowStart = bin(Timestamp, 5m) | sort by WindowStart desc`**

#### Explanation

The query first filters the input to error rows, then uses `summarize` with `count()` to aggregate records. The `bin(Timestamp, 5m)` expression creates explicit five-minute datetime buckets, and grouping by both `DeviceId` and the binned timestamp produces one count per device per window.

#### Why the other choices are incorrect

- **A:** This filters before aggregation, uses an explicit five-minute `bin`, groups by device and time bucket, and sorts the newest windows first.
- **B:** This summarizes before filtering on `Severity`, so the row-level severity column is no longer available and the result is not time-windowed.
- **C:** This creates a binned timestamp but only projects rows; it does not count events per device per window.
- **D:** Grouping by raw `Timestamp` creates a group for each exact timestamp value rather than five-minute windows.
