#### Question

A Fabric notebook creates a PySpark dataframe named `sales_curated_df` with `Year`, `Month`, and sales metric columns. Downstream jobs frequently read one month at a time from lakehouse files. You need to overwrite the curated file output and organize the files so Spark can prune folders by `Year` and `Month`. Which code should you use?

#### Choices

- A. `sales_curated_df.write.partitionBy("Year", "Month").mode("overwrite").parquet("Files/sales_curated")`
- B. `sales_curated_df.groupBy("Year", "Month").count().parquet("Files/sales_curated")`
- C. `sales_curated_df.write.mode("overwrite").parquet("Tables/sales_curated").partitionBy("Year", "Month")`
- D. `sales_curated_df.createOrReplaceTempView("Files/sales_curated")`

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. `sales_curated_df.write.partitionBy("Year", "Month").mode("overwrite").parquet("Files/sales_curated")`**

#### Explanation

The Spark dataframe writer uses `partitionBy` before the write action to create a folder hierarchy for the partition columns. Writing to `Files/sales_curated` as Parquet with overwrite mode stores file output while organizing it by `Year` and `Month`, which supports pruning when downstream reads target a specific month.

#### Why the other choices are incorrect

- **A:** This writes Parquet files and creates a multi-column partition hierarchy for the stated read pattern.
- **B:** `groupBy(...).count()` changes the dataset into aggregate counts and does not write the original curated sales rows.
- **C:** `parquet(...)` is the write action, so chaining `partitionBy` after it is not the supported writer pattern; `Tables/` is also the managed table area rather than the requested file output.
- **D:** A temporary view exists only for the current Spark session and does not create partitioned files for downstream jobs.
