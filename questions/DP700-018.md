#### Question

A PySpark notebook has transformed raw order rows into a dataframe named `orders_curated_df`. Analysts need to query the result later with Spark SQL, and the data should be stored as a managed Delta table in the lakehouse Tables area. Which code should you use?

#### Choices

- A. `orders_curated_df.write.format("delta").saveAsTable("orders_curated")`
- B. `orders_curated_df.createOrReplaceTempView("orders_curated")`
- C. `orders_curated_df.write.mode("overwrite").parquet("Files/orders_curated")`
- D. `display(orders_curated_df)`

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. `orders_curated_df.write.format("delta").saveAsTable("orders_curated")`**

#### Explanation

The Spark catalog can persist relational tables for Spark SQL queries. In Fabric, saving a dataframe with `format("delta").saveAsTable(...)` creates a managed table whose data is stored in the lakehouse Tables location. This satisfies the requirement for a later-queryable managed Delta table.

#### Why the other choices are incorrect

- **A:** This writes the dataframe as a Delta table in the Spark catalog, which makes it available for later Spark SQL queries.
- **B:** A temporary view is removed at the end of the current Spark session, so it does not meet the persistence requirement.
- **C:** Writing Parquet files under Files stores files, not a managed Spark catalog table in the Tables area.
- **D:** `display` renders the dataframe in the notebook output but does not save data or create a queryable table.

</details>
