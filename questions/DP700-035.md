#### Question

After publishing an eventstream that writes the Bicycles sample stream to an Eventhouse destination, a data engineer needs to confirm that recent events are arriving and review details about the captured event activity. Which action should the engineer take in the eventstream editor?

#### Choices

- A. Select the Eventhouse destination node, review the Data preview pane, refresh it after a few minutes, and inspect the Data insights tab
- B. Open the workspace settings page and check whether the workspace license mode is Trial, Premium, or Fabric
- C. Query the semantic model refresh history from Monitoring hub
- D. Create a new Eventhouse destination with a different table name each time the stream is checked

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Select the Eventhouse destination node, review the Data preview pane, refresh it after a few minutes, and inspect the Data insights tab**

#### Explanation

The lab validates the active Eventhouse destination from the eventstream canvas. After publishing and waiting for the destination to become active, the workflow selects the destination node, uses Data preview to inspect recent ingested rows, refreshes the preview after the perpetually running stream adds more data, and uses Data insights to review captured event details.

#### Why the other choices are incorrect

- **A:** This checks the live Eventstream destination and the event activity details directly in the canvas.
- **B:** Workspace capacity is required to use Fabric items, but it does not show whether the destination is actively receiving events.
- **C:** Semantic model refresh history monitors Power BI model refreshes, not Eventstream destination ingestion activity.
- **D:** Creating replacement destinations would change the ingestion topology and is unnecessary for monitoring whether the existing destination is active.
