#### Question

A Spark Job Definition continuously writes Event Hubs data to a Delta table. After the job restarts, it must recover its streaming progress instead of beginning again with no record of prior work. Which configuration is essential on the streaming write?

#### Choices

- A. Set a persistent, job-specific `checkpointLocation` in the lakehouse
- B. Change `outputMode` to `complete` and omit a checkpoint
- C. Run `VACUUM` on the target before every microbatch
- D. Create a deployment rule for the target table

<details>
<summary>Answer and explanation</summary>

#### Correct answer

**A. Set a persistent, job-specific `checkpointLocation` in the lakehouse**

#### Explanation

The Fabric Structured Streaming pattern stores checkpoint information in the lakehouse by setting `checkpointLocation` on `writeStream`. The checkpoint preserves the stream's progress and state so the job can recover across restarts.

#### Why the other choices are incorrect

- **A:** A durable checkpoint is the mechanism that records streaming progress for recovery.
- **B:** Output mode controls how results are written; it does not replace durable checkpoint state.
- **C:** `VACUUM` removes unreferenced Delta files and is unrelated to preserving streaming offsets or recovery state.
- **D:** Deployment rules configure stage-specific item settings and do not checkpoint a running stream.
