# DP-700 Anki Export

Maintain `exports/anki/dp700-anki-import.tsv` as a generated study artifact. Do not hand-edit the generated TSV when a source file or generator change is appropriate.

## Invoke This Skill

Use this skill explicitly when the work is Anki-focused:

```text
$dp700-anki-export
```

Example prompts:

- `$dp700-anki-export Add these recall cards to the DP-700 Anki source and regenerate the import file.`
- `$dp700-anki-export Check whether my new Anki cards are unique before I import them.`
- `$dp700-anki-export Regenerate the Anki TSV and report duplicate skips.`

## Source Order

The export is generated by `apps/quiz/scripts/export-anki.mjs` from these sources:

1. Canonical question-bank files: `questions/DP700-*.md`
2. Curated PDF concept cards embedded in `export-anki.mjs`
3. CSV question sets listed in `csvQuestionSetPaths` in `export-anki.mjs`
4. Consolidated review cards from `files/DP-700 Consolidated Review.txt`

Keep authored question-bank content in `questions/DP700-*.md`. Keep `questions/tracker/question_bank.md` free of full question text, answers, and explanations.

## Anki-Only Cards

For Anki-only study cards, use `files/DP-700 Consolidated Review.txt` as the source. Do not create or edit `questions/DP700-*.md` unless the user wants a full source-verified practice question for the quiz bank.

Add one tab-separated card per line:

```text
Front question<TAB>Back answer with explanation and source
```

Keep the front stable after import. Anki uses the first field to match existing notes during text import, so changing the front can create a new note instead of updating the old one.

Anki-only cards can be added from authoritative internet validation, local `files/` sources, or both. Use original wording and include the supporting link in the card back. Do not copy live-exam, recalled, commercial practice-test, or dump-style content into the cards.

## Source Correctness Validation

Use this workflow when adding or auditing Anki cards for technical correctness:

1. Search local sources under `files/`, relevant `questions/DP700-*.md`, and `apps/quiz/scripts/export-anki.mjs`.
2. Search authoritative internet sources, prioritizing Microsoft Learn, the official DP-700 study guide, and Microsoft Fabric documentation.
3. Add or update a card only when the answer is supported by the source. Put the source link directly in the card back.
4. For PDF text or rendering, use `apps/api/.venv/bin/python`.

When reporting an audit, classify findings as `Verified`, `Unsupported`, `Stale or ambiguous`, or `Needs source`.

## Human-in-the-Loop Source Intake

The user may place candidate study material under `files/` for review. Treat those files as user-provided reference material, not automatically trusted source material.

For Anki cards:

- Use the user-provided file to identify topics, gaps, or concepts.
- Verify technical correctness against Microsoft Learn, the official DP-700 study guide, or Microsoft Fabric documentation.
- Write cards in original wording.
- Include the authoritative Microsoft source link in the card back.
- Do not copy question text, answer choices, or explanations from exam dumps, recalled exam material, or commercial practice-test sites.

## Update Workflow

1. Inspect the requested source:
   - For canonical DP-700 questions, read `questions/tracker/question_bank.md` and relevant `questions/DP700-*.md`.
   - For consolidated review cards, inspect `files/DP-700 Consolidated Review.txt`.
   - For curated PDF concept cards, edit the `pdfCards` array and `sourceUrlByPdfPath` map in `apps/quiz/scripts/export-anki.mjs`.
   - For CSV question-set cards, inspect the source CSV under `files/` and the `csvQuestionSetPaths` list in `apps/quiz/scripts/export-anki.mjs`.
2. Preserve source ownership:
   - Add exam-style, source-verified scenarios to `questions/DP700-*.md`, not directly to Anki.
   - Add quick recall or study-note cards to `files/DP-700 Consolidated Review.txt`.
   - Add PDF-derived concept cards in `export-anki.mjs` only when they are original summaries, not copied assessment items.
   - Add CSV question-set files to `files/` only when they are original, source-verified, and intended to be generated into the main Anki TSV.
   - Add internet-validated recall cards to `files/DP-700 Consolidated Review.txt` when the user wants Anki-only study cards rather than quiz-bank scenarios.
3. Regenerate from WSL Ubuntu:

```bash
cd apps/quiz
nvm use
npm run export:anki
```

4. Validate:

```bash
npm run validate:questions
cd ../..
git diff --check
awk 'BEGIN{bad=0; cards=0} /^#/ {next} { cards++; if (NF != 3) { print "bad row", NR, "fields", NF; bad=1 } } END{ print "card rows", cards; if (bad) exit 1 }' FS='\t' exports/anki/dp700-anki-import.tsv
```

Report the card counts from `npm run export:anki`, any skipped duplicates, and whether validation passed.

## Uniqueness Policy

Treat the Anki card front as the import identity. The generator normalizes fronts by stripping source prefixes, HTML tags, punctuation, and case, then keeps the first card and skips later duplicates.

When adding new cards:

- Search before editing: `rg -n "distinctive phrase from the front" questions files apps/quiz/scripts/export-anki.mjs exports/anki/dp700-anki-import.tsv`
- Prefer improving an existing card when the new card tests the same fact with nearly the same wording.
- Add a new card only when it tests a distinct decision, command, concept boundary, or scenario.
- Keep duplicate-but-useful variations only when they test genuinely different reasoning; rewrite the front so the distinction is explicit.

If `npm run export:anki` reports skipped duplicate fronts, inspect whether the source card should be removed, merged, or rewritten. Do not silence the duplicate output without resolving the underlying overlap.

## Import Format

The generated TSV uses:

```text
#separator:tab
#html:true
#notetype:Basic
#deck:DP-700
#tags column:3
```

Each card row must have exactly three tab-separated fields: front, back, and tags. The back should include visible source context and links when available.
