# DP-700 Anki Weak Points

Assess weak DP-700 topics from an Anki `.apkg` export that includes scheduling information. Keep Anki-derived progress in `docs/quiz_progress_anki.md`; do not write learner performance into `questions/`, `questions/tracker/question_bank.md`, or generated Anki import TSV files.

## Inputs

- Default deck export path: `files/DP-700 Consolidated Review.apkg`
- Alternate export path: any user-provided `.apkg`
- Progress report path: `docs/quiz_progress_anki.md`
- Helper script: `scripts/analyze_apkg.py`

The `.apkg` must be exported from Anki with scheduling information included. If scheduling data is missing, ask the user to re-export the deck with scheduling/review history.

## Workflow

1. Locate the `.apkg` file. Prefer the user-provided path; otherwise check `files/DP-700 Consolidated Review.apkg`.
2. Run the helper from the repository root:

```bash
python3 .agents/skills/dp700-anki-weak-points/scripts/analyze_apkg.py \
  "files/DP-700 Consolidated Review.apkg" \
  --output docs/quiz_progress_anki.md \
  --deck DP-700
```

3. If the helper reports that `collection.anki21b` requires Zstandard support, install `zstandard` in the WSL Python environment or ask the user for an older Anki export format that contains a readable `collection.anki2`.
4. Review the generated report. Prioritize:
   - cards with recent `Again` reviews,
   - cards with lapses,
   - cards with low success rate after at least two reviews,
   - DP-700 domains/topics with the highest aggregate weak score.
5. Recommend concrete follow-up:
   - review cards in Anki by tag,
   - run a `$dp700-quiz-bank` tracked quiz on the weak topic,
   - create more source-verified questions only when the question bank lacks coverage for that weak area.

## Interpretation Rules

- `Needs review`: latest review was `Again`, a recent `Again` exists, the card has lapses, or success rate is below 70% after at least two reviews.
- `Learning`: reviewed but not yet strong or needs-review.
- `Strong`: at least three reviews, success rate at least 85%, latest review was `Good` or `Easy`, and no recent `Again`.
- `Unreviewed`: no review history in the exported deck.

Treat Anki results as personal study signals, not exam score predictions.

## Safety Rules

- Do not expose or modify question-bank answer keys unless the user explicitly asks to inspect a question.
- Do not import exam dumps, recalled live-exam questions, or commercial practice-test content.
- Do not edit `exports/anki/dp700-anki-import.tsv`; it is generated by `$dp700-anki-export`.
- Preserve existing `docs/quiz_progress.md`; Anki-derived progress belongs in `docs/quiz_progress_anki.md`.
